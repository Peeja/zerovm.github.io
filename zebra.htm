ZeroVM and OpenStack® Swift

ZeroVM has been integrated into OpenStack Swift in a project code named ZeroCloud. ZeroCloud consists of a job manager which is installed at the proxy layer and a ZeroVM daemon which is installed on the storage nodes. The addition of ZeroVM converts Swift from a static object storage solution into a dynamic converged storage and compute platform. Users can manipulate their objects without ever removing them from the storage cluster leveraging the spare compute cycles on the storage nodes. In addition to basic object manipulations users can create complex pipelines and even perform full map–reduce operations, all without ever extracting their data from the object storage cluster.

The Zebra Environment

What is Zebra?

Zebra is a ZeroCloud playground provided by Rackspace to the ZeroVM community. Zebra provides the ZeroVM community a place to experimente with the technology and a place to test new features and functionality at scale to help accelerate development. This is also the perfect environment on which to develop and test ZeroVM applications. Zebra can be found at https://zebra.zerovm.org but you will need to be granted access to the environment before you can use it (see below).
